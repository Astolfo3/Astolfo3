## 🌸 about me

> *"Systems programming shouldn't look like a terminal from 1985. Performance and personality are not mutually exclusive."*

I'm **Astolfo** — a low-level systems engineer who believes code can be both **performant** and **beautiful**. I write Rust, craft cute tooling, and design pastel UIs for the terminal. Inspired by the fearless, cheerful energy of Astolfo (Rider of Black, *Fate/Apocrypha*), I bring that same spirit to every project: **fearless, colorful, unapologetically me**.

<br/>

## what i'm about

| pillar | meaning |
|--------|---------|
| **safe** | memory-safe, type-driven, reliable systems |
| **fast** | zero-cost abstractions, minimal overhead, maximal throughput |
| **🌸 cute** | readable code, pretty terminals, welcoming interfaces |

<br/>

## stack & tools

| domain | tech |
|--------|------|
| **language** | Rust (primary), C, Python (tooling) |
| **editor** | Neovim — pastel config, pink everything |
| **terminal** | Alacritty with blur & custom colorscheme |
| **shell** | Fish — sparkly prompt, all lowercase, lots of `:3` |
| **OS** | Arch Linux |
| **font** | Nerd Fonts — ligatures, glyphs, the works |

<br/>

## philosophy

1. **readability first** — code is for humans first, machines second. if it's not readable, it's not done.
2. **adorable by default** — UIs, terminals, docs, repos — everything deserves a touch of cute.
3. **zero-cost personality** — cuteness should never cost performance. you can have both.
4. **fearless prettiness** — bold colors, soft gradients, and thoughtful aesthetics belong everywhere, even in low-level spaces.
5. **stay approachable** — systems programming is intimidating enough. let's make it welcoming.
